# FPDev v2.1.0 Release Owner Checkpoints

## Goal

Close FPDev `v2.1.0` with a public `GitHub Actions` release-proof flow that produces publish-ready artifacts for Linux, Windows, and macOS.

This document is intentionally narrow:
- it does not reopen feature work
- it does not reopen broad code review
- it defines the primary public CI proof bundle and the local fallback procedure when a transcript must be re-recorded

## Primary Release Proof

The primary publish-time evidence comes from `GitHub Actions`.

Expected public CI artifacts:
- `release-asset-linux-x64`
- `release-asset-windows-x64`
- `release-asset-macos-x64`
- `release-asset-macos-arm64`
- `release-acceptance-logs`
- `owner-proof-windows-x64`
- `owner-proof-macos-x64`
- `owner-proof-macos-arm64`
- `release-ready-bundle`

`release-ready-bundle` is the preferred handoff artifact before publish. It should contain:
- the four planned release assets
- `SHA256SUMS.txt`
- `RELEASE_EVIDENCE.md`
- Linux acceptance logs
- standardized owner-proof transcripts

## Automated Gate

Run the local Linux release lane first when validating outside CI:

```bash
bash scripts/release_acceptance_linux.sh
```

If a clean-host Linux install check is required before publishing, use the optional network-gated mode:

```bash
bash scripts/release_acceptance_linux.sh --with-install
```

## Planned Release Assets

| Asset | Purpose |
|------|---------|
| `fpdev-linux-x64.tar.gz` | Linux x86_64 binary release |
| `fpdev-windows-x64.zip` | Windows x64 binary release |
| `fpdev-macos-x64.tar.gz` | macOS Intel binary release |
| `fpdev-macos-arm64.tar.gz` | macOS Apple Silicon binary release |
| `SHA256SUMS.txt` | Release checksum manifest |

## Publish Sequence

1. Download `release-ready-bundle` from `GitHub Actions` for the release commit.
2. If the bundle must be reconstructed locally, download `release-asset-linux-x64`, `release-asset-windows-x64`, `release-asset-macos-x64`, `release-asset-macos-arm64`, `release-acceptance-logs`, `owner-proof-windows-x64`, `owner-proof-macos-x64`, and `owner-proof-macos-arm64`.
3. When rebuilding the asset directory manually, use `python3 scripts/package_release_assets.py --output-dir <asset-dir> --data-dir src/data --linux-bin <linux-binary> --windows-bin <windows-binary> --macos-x64-bin <macos-x64-binary> --macos-arm64-bin <macos-arm64-binary> --require-planned-assets`.
4. Generate `SHA256SUMS.txt` with `python3 scripts/generate_release_checksums.py <asset-dir> --require-planned-assets`.
5. Generate the release evidence handoff with `python3 scripts/generate_release_evidence.py --baseline-summary logs/release_acceptance/<baseline-run>/summary.txt --asset-dir <asset-dir> --owner-proof-dir <owner-proof-dir> --output <asset-dir>/RELEASE_EVIDENCE.md`.
6. Add `--install-summary logs/release_acceptance/<install-run>/summary.txt` only when the optional Linux install lane was recorded.
7. Publish only after the public CI bundle or its local reconstruction contains assets, checksums, release evidence, and owner-proof transcripts.

## Local Fallback Recorders

Use the local fallback recorders only when `GitHub Actions` artifacts are missing, a platform smoke needs to be re-run, or a maintainer must regenerate standardized transcripts outside CI.

### Windows x64 Fallback

#### Setup

```powershell
Expand-Archive -Path .\fpdev-windows-x64.zip -DestinationPath .\fpdev-windows-x64
```

#### Recorded Smoke Command

Use `scripts/record_owner_smoke.ps1` so the fallback produces the standard transcript file `windows-x64-owner-smoke.txt`.
The recorder wraps the shared smoke suite in `scripts/cli_smoke.ps1`.

```powershell
pwsh ./scripts/record_owner_smoke.ps1 -Lane windows-x64 -ExecutablePath .\fpdev-windows-x64\fpdev.exe -OutputDir .\owner-proof
```

### macOS x64 / arm64 Fallback

#### Setup

```bash
tar -xzf fpdev-macos-<arch>.tar.gz
```

Replace `<arch>` with `x64` or `arm64`.

#### Recorded Smoke Command

Use `scripts/record_owner_smoke.sh` so the fallback produces `macos-x64-owner-smoke.txt` or `macos-arm64-owner-smoke.txt`.
The recorder wraps the shared smoke suite in `scripts/cli_smoke.sh`.

```bash
bash ./scripts/record_owner_smoke.sh macos-<arch> ./fpdev-macos-<arch>/fpdev ./owner-proof
```

## Sign-Off Ledger

| Lane | Primary proof | Manual fallback |
|------|---------------|-----------------|
| Linux automated acceptance | `release-ready-bundle/release-acceptance-logs/.../summary.txt` | `logs/release_acceptance/<baseline-run>/summary.txt` |
| Linux isolated install lane | optional entry in `RELEASE_EVIDENCE.md` | `logs/release_acceptance/<install-run>/summary.txt` |
| Windows x64 asset smoke | `owner-proof-windows-x64/windows-x64-owner-smoke.txt` | `owner-proof/windows-x64-owner-smoke.txt` |
| macOS x64 asset smoke | `owner-proof-macos-x64/macos-x64-owner-smoke.txt` | `owner-proof/macos-x64-owner-smoke.txt` |
| macOS arm64 asset smoke | `owner-proof-macos-arm64/macos-arm64-owner-smoke.txt` | `owner-proof/macos-arm64-owner-smoke.txt` |

## Release Exit Criteria

- `release-ready-bundle` exists in public `GitHub Actions` results for the release commit, or an equivalent local reconstruction has been produced.
- Release assets match the filenames in this document.
- `SHA256SUMS.txt` is published with the release.
- `RELEASE_EVIDENCE.md` records Linux automated acceptance and any optional install proof that was collected.
- Windows x64, macOS x64, and macOS arm64 owner-proof transcripts are available from public CI or the documented fallback procedure.
