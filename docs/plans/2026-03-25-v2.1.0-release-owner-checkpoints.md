# FPDev v2.1.0 Release Owner Checkpoints

## Goal

Close FPDev `v2.1.0` with one automated Linux acceptance lane and explicit owner-run checkpoints for the platforms that cannot be proven in this workspace.

This document is intentionally narrow:
- it does not reopen feature work
- it does not reopen broad code review
- it exists to finish release close-out with clear handoff points

## Automated Gate

Run the local Linux release lane first:

```bash
bash scripts/release_acceptance_linux.sh
```

If a clean-host Linux install check is required before publishing, use the network-gated mode:

```bash
bash scripts/release_acceptance_linux.sh --with-install
```

## Planned Release Assets

| Asset | Purpose |
|------|---------|
| `fpdev-linux-x64.tar.gz` | Linux x86_64 binary release |
| `fpdev-windows-x64.zip` | Windows x64 binary release |

| `fpdev-macos-arm64.tar.gz` | macOS Apple Silicon binary release |
| `SHA256SUMS.txt` | Release checksum manifest |

## Publish Sequence

1. Run `bash scripts/release_acceptance_linux.sh` and keep the log directory path.
2. Package the prepared platform binaries with `python3 scripts/package_release_assets.py --output-dir <asset-dir> --data-dir bin/data --linux-bin <linux-binary> --windows-bin <windows-binary> --macos-x64-bin <macos-x64-binary> --macos-arm64-bin <macos-arm64-binary> --require-planned-assets`.
3. Generate `SHA256SUMS.txt` from the packaged assets with `python3 scripts/generate_release_checksums.py <asset-dir> --require-planned-assets`.
4. Generate a release-evidence handoff with `python3 scripts/generate_release_evidence.py --baseline-summary logs/release_acceptance/<baseline-run>/summary.txt --asset-dir <asset-dir> --owner-proof-dir <owner-proof-dir> --output <asset-dir>/RELEASE_EVIDENCE.md`. If the network-gated install lane was executed, append `--install-summary logs/release_acceptance/<install-run>/summary.txt`.
5. Upload assets, checksums, and `RELEASE_EVIDENCE.md` to the `v2.1.0` GitHub release draft.
6. Execute the owner checkpoints below on Windows and macOS.
7. Publish only after every mandatory checkpoint has an owner, date, and evidence reference.

## Windows x64 Owner Checkpoint

### Setup

```powershell
Expand-Archive -Path .\fpdev-windows-x64.zip -DestinationPath .\fpdev-windows-x64
```

### Recorded Smoke Command

Use the owner smoke recorder so the checkpoint produces the standard transcript file `windows-x64-owner-smoke.txt`.
The recorder wraps the shared smoke suite in `scripts/cli_smoke.ps1`.

```powershell
pwsh ./scripts/record_owner_smoke.ps1 -Lane windows-x64 -ExecutablePath .\fpdev-windows-x64\fpdev.exe -OutputDir .\owner-proof
```

### Expected Result

- every command exits `0`
- output is readable English CLI output
- `fpc --help` shows the FPC namespace surface
- `fpc list --all` includes `3.2.2`

## macOS x64 / arm64 Owner Checkpoint

### Setup

```bash
tar -xzf fpdev-macos-<arch>.tar.gz
```

Replace `<arch>` with `x64` or `arm64`.

### Recorded Smoke Command

Use the owner smoke recorder so the checkpoint produces `macos-x64-owner-smoke.txt` or `macos-arm64-owner-smoke.txt`.
The recorder wraps the shared smoke suite in `scripts/cli_smoke.sh`.

```bash
bash ./scripts/record_owner_smoke.sh macos-<arch> ./fpdev-macos-<arch>/fpdev ./owner-proof
```

### Expected Result

- every command exits `0`
- Gatekeeper/quarantine issues are resolved before sign-off
- output is readable English CLI output
- `fpc list --all` includes `3.2.2`
